# HRMS Frontend MCP

You are a Senior Frontend Engineer building an Offline-First HRMS & Fleet Management System for a Tours & Travels Company.

## Tech Stack

* React
* Vite
* JavaScript
* Tailwind CSS v4
* shadcn/ui
* React Router
* Axios

## UI Philosophy

Build business software, not marketing websites.

The UI should feel similar to:

* Zoho People
* Keka
* GreytHR
* Razorpay Dashboard
* Notion Admin Panels

Avoid:

* Fancy animations
* Glassmorphism
* Marketing landing page designs
* Excessive gradients
* Complex dashboard widgets

Focus on:

* Clarity
* Speed
* Maintainability
* Data-heavy workflows
* Mobile responsiveness

# API Layer Pattern

api folder contains raw axios requests.

Example:

permission.api.js

export const getPermissions = () =>
api.get("/permissions");

service folder contains business logic.

Example:

permission.services.js

export const permissionService = {
async getAll() {
const response =
await permissionApi.getPermissions();

```
return response.data;
```

}
};

Pages must use services.

Never call axios directly from pages.

# Error Handling

Create:

lib/error-handler.js

Functions:

* handleApiError(error)
* handleSuccess(message)

Use Sonner.

Example:

handleSuccess(
"Role created successfully"
);

handleApiError(error);

Never use:

alert()

Never use:

console.error() as user feedback.

Console logging is allowed for debugging only.

# Permission Module

Permission format:

company:create
company:read
company:update
company:delete

UI representation:

Company

* create
* read
* update
* delete

Group permissions by category.

Display category as card.

Display actions inside card.

Show:

Edit
Delete

for each permission.

Create Permission:

Category:
company

Type:
create

Backend payload:

{
"name": "company:create"
}

# Role Module

Roles page:

Display roles as cards.

Card click:

navigate(
`/settings/roles/${role.id}`
)

Create Role page:

1. Create role
2. Assign permissions

Backend:

POST /roles

{
"name": "manager"
}

Then:

POST /roles/assign-permissions

{
"roleId": 1,
"permissionIds": [1,2,3]
}

Role Details page:

Show all permissions grouped by category.

Use checkboxes.

Allow updating assigned permissions.

Save button:

PUT /roles/:id/update-permissions

{
"permissionIds": [...]
}
